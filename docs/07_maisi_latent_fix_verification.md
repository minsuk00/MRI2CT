**Source HTML:** _html/02a_maisi_latent_fix_verification.html

**TL;DR:** The dynamic_infer fix (whole-volume encode if it fits the ROI, else ROI clamped to the volume, never pad beyond input) removes the VAE encoder's GroupNorm-over-zero-padding artifact. Brain decode(encode(GT)) soft-tissue error is now -26 HU on average (was +39...+85 with the oversized-ROI padding); no brightening. Follow-up to the MAISI VAE audit (docs/06).

Fix: dynamic_infer applied in encode_all_volumes.py and trainer._encode_sliding_window, ROI [320,320,160] (matches NV-Generate-CTMR). This is the VAE round-trip on ground-truth CT with the patched encoder (the exact function used to regenerate the training latents), so it is the model's achievable ceiling per region.

## Brain at the clinical [-100,100] HU window

The window that washed out white before. With the fixed encode, brain reconstructs as mid-gray: the residual is a mild ~-16...-30 HU softening (VAE compression), not the +85 HU brightening.

## Mechanism

The corrupted brightening was caused by the VAE encoder's 13 GroupNorm layers (decoder has 13) normalizing over a volume flooded with zero-padding; dynamic_infer removes the padding so the encoder GroupNorm sees only real anatomy.

Generated by `~/vae_audit/verify_and_02a.py`. Encoder is variational (sampled), so values vary ~±a few HU per run. Companion to the retracted/corrected 02_maisi_vae_brain_hu_audit.html.
