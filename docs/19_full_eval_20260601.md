## Track A: amix-clip [-1024, 1024] (hu_range 2048)

- Body-masked metrics (first 3 columns) are the PRIMARY, fair comparison: voxels outside the body mask zeroed, metric over the volume.
- Hard Bone Dice and Hard Dice (mean over all 11 foreground organ classes) via the Baby-UNet teacher (`ckpt/seg_baby_unet/seg_baby_unet_epoch_749.pth`), computed on argmax labels following dice-score-3d convention: per class, pred & GT both absent -> 1.0, exactly one absent -> 0.0, else 2|A intersect B|/(|A|+|B|). ## Per-region (Track A)

- Error bars = mean +/- std over each region's val subjects. Hard Dice exists only in Track A (computed in amix-clip space).
- Read per-region Dice via Bone Dice: the per-region all-class Hard Dice is region-confounded. The all-class mean averages over all 11 teacher organ classes, but a limited FOV (e.g. head/neck) anatomically contains only ~6. The GT seg (`ct_seg.nii`) carries spurious labels for absent torso organs (e.g. head/neck case tagged with ~3,000 voxels "breast implant" plus stray abdominal-cavity / pericardium / mediastinum), which the synthesis correctly does not reproduce, so those classes score ~0 and drag the mean down (head/neck amix all-class 0.49 vs Bone 0.79). Metric + GT-label artifact, NOT a synthesis deficit: present classes score well (head/neck Brain 0.96, Bones 0.81, Muscle 0.77), effect hits every model equally. Hard Bone Dice (a class present in every region) is the trustworthy per-region Dice signal.

Worked example: head/neck subject 1HNC007 (amix prediction), per-class Hard Dice. Five spurious / out-of-FOV classes (breast implant, abdominal cavity, pericardium, thoracic cavity, mediastinum) score ~0 and pull the 11-class mean to 0.394:

## Inference time

- amix / unet / maisi re-measured with CUDA synchronization on 10 representative subjects (2 per region, brain -> thorax). An earlier wall-clock without sync under-counted these fast models (measured: unet ~11x, amix ~2.9x, maisi ~1.6x).
- Diffusion models are sync-invariant (GPU time dominated by the multi-step sampling loop), so cWDM and MC-DDPM use full-207 generation means as-is.
- Times scale strongly with volume size; treat as order-of-magnitude.

## Training configuration: original (paper/code) vs ours

"Paper says" / "official code default" / "our run" kept separate. Diffusion baselines (esp. MAISI, cWDM) are snapshots, often under-trained and adapted from the originals.

### amix & unet (reference, 1.00x)

| Param | amix (anatomix translator) | unet (baseline) |
| --- | --- | --- |
| Architecture | Frozen Anatomix v1_4 feature extractor -> U-Net translator (sigmoid) | Plain 3D U-Net (sigmoid) |
| Patch / batch | 128^3 patch, eff. batch 8 (4x2 patches) | 128^3 patch, batch 8 (x1 patch) |
| Schedule | 800 epochs x 500 steps = 3,200,000 samples (ckpt ep799) | 800 epochs x 500 steps = 3,200,000 (ckpt ep799) |
| CT / MR norm | CT clip [-1024,1024]->[0,1]; MR min-max | same |
| Loss | L1 + SSIM (+ optional teacher/bone Dice) | L1 + SSIM |
| Inference | sliding-window 256^3, overlap 0.25 | same |

### MC-DDPM / MC-IDDPM (1.25x of our 3.2M budget)

Pan et al., Med Phys 2024 (3D Swin-transformer denoising diffusion). Official repo: shaoyanpan/Synthetic-CT-...-diffusion-model. "MC-DDPM" = "MRI-to-CT DDPM" per the paper, NOT "Monte Carlo." Original trained on a tiny institutional set (28 brain / 20 prostate volumes), so our SynthRAD run (427 train vols, 4.0M slabs) is far MORE, not less, trained.

| Param | Paper | Official code (notebook) | Our run |
| --- | --- | --- | --- |
| Architecture | Swin-Vnet transformer | SwinVITModel (C=64, mult 1,2,3,4) | identical kwargs |
| Slab / batch / data | 64^3x4 (brain), 128^2x4 (prostate); per-anatomy models | (64,64,4), bs 4 x2 patches | (128,128,4) slab, bs 4x2; one model on ALL 5 regions (center_wise 427 train) |
| Diffusion T / schedule | 1000 | 1000, linear, learn_sigma=True | 1000, linear, learn_sigma=True |
| Optimizer / LR | AdamW, 3e-5, wd 1e-5 | AdamW, 2e-5, wd 1e-4 (differs from paper) | AdamW, 3e-5, wd 1e-5 (follows paper text) |
| Total training budget | 500 epochs on 28 brain / 20 prostate vols -> ~28,000 / 32,000 patch-presentations (~7-8k steps) | same (one pass/epoch over per-vol crops) | fixed 500 steps/epoch; eval ckpt ep1000 = 4,004,000 slabs (full run ~12M). Far MORE than original (427 train vols). |
| CT / MR norm | CT clip [-1024,1650]->[-1,1]; MR min-max [-1,1] | same | same |
| Sampling steps | 50 respaced steps, 50% overlap | respaced 50, overlap 0.5 | 50 respaced (ancestral DDPM, CLI mislabels it "DDIM"), overlap 0.25 |
| Test-time MC averaging | N=5 runs averaged (paper text) | N=1 (released notebook calls sampler ONCE per subject, no averaging) | mc_runs=1 (matches released code, NOT the paper's N=5) |

Divergences: paper claims 5-run MC test-time averaging but official released code does not implement it (single draw); our mc_runs=1 matches the original code, diverging only from paper prose. Other diffs: 0.25 vs 0.5 SW overlap; the 4-slice (128x128x4) slab gives little long-axis context on larger volumes (thorax/head-neck); single model on all 5 regions vs paper's per-anatomy models; sample-budget-matched ep1000 cut. No EMA in paper or code. (Verified against original inference notebook.)

### MAISI (0.26x budget, under-trained)

NVIDIA MAISI (v1 DDPM / v2 rectified-flow). We FREEZE the pretrained CT VAE + RFlow diffusion U-Net and train only a ControlNet, re-wired from 8-ch seg-mask conditioning to 1-ch MR (custom MR->CT adaptation, no upstream equivalent).

| Component / Param | Paper (v2 RFlow) | Official code (rflow-ct) | Our run |
| --- | --- | --- | --- |
| VAE | 3D VAE-GAN, latent 4-ch, 8x downsample | pretrained autoencoder_v1.pt | frozen (inherited) |
| Diffusion U-Net | RFlow, T=1000, 64xA100 ~3 wk | pretrained diff_unet_3d_rflow-ct.pt | frozen (inherited) |
| ControlNet cond. | seg mask (127 cls, 8-ch) | 8-ch seg mask | 1-ch MR image (our adaptation) |
| ControlNet LR | 5e-5, poly decay | 1e-5 (JSON) | 1e-4, scheduler disabled (constant) |
| Batch / volume | bs 1, 8xA100 | bs 1 | bs 1 x accum 8 = eff. 8, full padded volume, 1xA40 |
| Loss | RFlow velocity + tumor-weighted x100 + region contrastive | weighted_loss=100, contrastive | plain L1 on RFlow velocity (dropped tumor/contrastive, irrelevant to MR->CT) |
| Aug. | heavy (bias field, Gibbs, contrast, flips...) | - | none (static pre-encoded latents) |
| Train budget | ~60 epochs / ~2 days | n_epochs 100 | eval ckpt ep2110 = 844,400 samples (still training) |
| Inference | 30 RFlow steps | 30 steps, SW decode overlap 0.4 | 30 steps, SW decode overlap 0.4 |

Divergences: MR->CT ControlNet is our own (no upstream MR->CT recipe); simplified loss (plain L1); constant lr 1e-4; no augmentation; 1xA40 vs 8xA100; heavily under-trained (0.26x). Matches originals on CT/MR norm, RFlow scheduler, 30 inference steps, decode overlap 0.4.

Friedrich et al. 2024 (conditional wavelet diffusion; WDM-3D line). Official repo pfriedri/cwdm targets BraTS (3 MR->1 MR); we adapt to SynthRAD MR->CT (single input).

| Diffusion T / schedule | 1000, linear, predict_xstart, learn_sigma=False | same | same |
| Optimizer / LR / EMA | Adam, 1e-5 | Adam 1e-5, EMA 0.9999 | Adam 1e-5, EMA 0.9999, constant LR |
| Train iterations | 1.2M | ITERATIONS=1200 (x1000 = 1.2M) | ~354,000 (~0.30x target), still training; bs 1 -> ~354k samples |

Divergences: heavily under-trained (~354k vs 1.2M target, ~0.11x our other baselines); DDIM-100 vs upstream 1000-step DDPM; in_channels 16 vs 32 (correct single-input adaptation). Eval checkpoint pinned to permanent milestone synthrad_354000.pt.

## Methods & choices

- Volume generation: amix/unet/maisi/mcddpm/cwdm regenerated on the 207 val subjects via each model's `validate.py` (sbatch/fulleval_*.sh), writing per-subject HU `sample.nii.gz` + `target.nii.gz` + `validate_metrics.txt` to `raw/<model>/` (arrays under `raw/<model>/shard_*_of_8/`). koalAI NOT regenerated; fold-0 HU predictions already exist at `.../fold_0/validation_revert_norm/<subj>.mha` (207 subjects, full de-normalized HU, bone >1024 preserved), converted to NIfTI in the unified dir.
- Two metric yardsticks (Track A / Track B), each with full-volume and body-masked variants:
  - Track A (amix-clip [-1024,1024], hu_range 2048): `mae_hu`/`psnr`/`ssim` = full volume no mask (`compute_metrics`); `body_*` = zero-out style (voxels outside body mask -> 0, then metric over full zeroed volume, `compute_metrics_body`); `dice_score_all`/`dice_score_bone` = Baby U-Net teacher Hard Dice (argmax, dice-score-3d convention). `dice_score_all` = mean over 11 fg classes; `dice_score_bone` = class 5. (Switched from a prior soft probability Dice to hard/argmax to match standard eval convention.)
  - Track B (SynthRAD-native [-1024,3000], PSNR data_range 4024): `synthrad_mae`/`synthrad_psnr`/`ms_ssim` = body-voxels-only style (sum(|err|*mask)/mask.sum()), official SynthRAD/koalAI `ImageMetrics` convention. Body mask source: dataset `mask.nii` / koalAI origin `MASKS/<subj>.mha` (same mask koalAI used).
  - Avg inference time per volume = sum(`time_sec`)/207 per model (koalAI pre-generated -> n/a).
- Single common ground truth (verified): all models scored against ONE common GT, the raw full-HU dataset `ct.nii` (bone up to ~3000 HU), NOT each model's own saved `target.nii.gz`. Per-model targets are clipped differently (amix/unet [-1024,1024], MAISI [-1000,1000], MC-DDPM [-1024,1650], koalAI raw), which would make Track B unfair. Verified: clip(raw,-1024,1024) reproduces amix's saved target exactly (delta=0), so Track A unchanged (scorer matches amix's canonical validate.py to 4 dp), while Track B fairly penalizes capped models on real bone >1024. koalAI's origin TARGET == dataset `ct.nii` exactly (delta=0) and dataset mask == origin mask (100%).
- koalAI cross-check: `koalai_native` is OUR OWN earlier evaluation of koalAI (official SynthRAD `ImageMetrics` run on koalAI predictions in a prior session), NOT a koalAI-published result (the KoalAI paper reports pooled, 5-fold-ensemble numbers on different folds). "Track-B reproduces koalai_native exactly (delta=0, e.g. 1THB006=106.75)" is a self-consistency / regression check, not validation against any koalAI-reported number.

### Caveats for interpretation
- Under-training dominates any ranking. MAISI (0.26x) and cWDM (0.11x) are still training and far below the 3.2M budget; their rows are SNAPSHOTS, not converged quality. Do NOT read them as "diffusion is worse." The x3.2M budget travels with every results table.
- Track A erases koalAI's bone advantage twice (image metrics + teacher input both clipped to [-1024,1024]). Track A measures within-window/soft-tissue fidelity; bone-HU fidelity lives only in Track B. Don't cite Track-A bone Dice as evidence koalAI's bone is worse.
- Hard Bone Dice and Hard Dice are a segmentation-utility PROXY (Baby-UNet teacher trained on real CT in [-1024,1024]); cannot credit bone HU >1024. Hard Dice is region-confounded (all-class mean includes torso-only classes absent in limited FOV; absent-in-both scores 1.0 by convention) -> read alongside Hard Bone Dice. Whole-volume Hard Dice is unmasked.
- Sampling configs are each model's own default, not a matched budget: MAISI 30 steps, MC-DDPM DDIM 50 with mc_runs=1 (single draw, under-uses MC-DDPM's multi-sample design), cWDM DDIM 100. A low diffusion score may be under-training and/or sampling, not separable.
- Headline on macro (equal region weight) or per-region, not a single micro number: regions differ in size (brain 60, abdomen 52 vs pelvis 30) and OOD difficulty. Both micro & macro in overall.csv.
- Significance: with n=30-60/region, small gaps among the well-trained three (amix/unet/koalAI) may be noise; treat un-significant Wilcoxon gaps as ties.
- amix uses a frozen Anatomix extractor pretrained on external data: a fair system comparison, not an equal-data one.
- cWDM volume fix: validate.py originally saved sample/target in [0,1] (not HU); fixed to save HU, already-generated volumes converted in place (val*2048-1024). Verified: converted cWDM scores reproduce cWDM's own inline metric exactly (1ABB039 body-MAE 26.8). A guard in score_all_models now refuses any non-HU (normalized) sample.
