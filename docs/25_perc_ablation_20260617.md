# Anatomix-injection ablation

**Source HTML:** _html/perc_ablation_20260617.html
**Date:** 2026-06-17
**TL;DR:** Three variants sharing an L1 + Dice base differ only in where anatomix is injected: nowhere (unet-dice) vs as a training loss (unet-perc, NCC on frozen anatomix features) vs as the backbone (amix). At ep200/100k steps, unet-dice (no anatomix) has the best body MAE (macro 34.8HU); injecting anatomix as a loss or backbone does not improve body MAE here (macro 36.3 / 37.0HU) and is significantly worse per-subject (Wilcoxon p=8.2e-20 / 6.4e-25). Bone Dice is roughly tied between unet-dice and amix.

Latest snapshot of the perc_ablation series (docs/23 dated 2026-06-03, docs/24 dated 2026-06-09). This one shifts focus from perceptual-loss on/off to the broader question of where anatomix is injected. Center-wise validation split, 207 subjects, all at the 100k-step (epoch 200) parity checkpoint, Track-A metrics + Hard Dice, identical definitions to full_eval.

## Variants

- unet-dice (no anatomix) (nbn71048_ep200): plain U-Net, L1 + SSIM + Dice, anatomix used nowhere, n=207
- unet-perc (anatomix loss) (y5tqp2bt_ep200): plain U-Net, L1 + anatomix-NCC perceptual(0.1) + Dice, SSIM off, anatomix in the loss, n=207
- amix v1.4 (anatomix backbone) (827la6dp_ep200): anatomix-feature translator, L1 + SSIM + Dice (perceptual_w=0), anatomix in the architecture, n=207

All three share the L1 + Dice base and the same center-wise data/budget (ep200 ≈ 100k steps), and differ only in where anatomix is injected: nowhere -> training loss (NCC on frozen anatomix decoder features) -> model backbone (translator on frozen anatomix features, no perceptual loss). Caveat: unet-dice -> unet-perc swaps SSIM for the perceptual term (repo rule: perceptual on => ssim_w=0), so that step is a substitution, not a pure addition; amix keeps SSIM. Each variant is a separate training run, so a small part of any gap is run-to-run noise.
